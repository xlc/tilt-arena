# GitHub Workflow Sources

- GitHub Issues track ideas and work:
  https://docs.github.com/en/issues/tracking-your-work-with-issues
- Sub-issues break larger issues into tasks and preserve parent/child tracking:
  https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/adding-sub-issues
- Issues can be linked and closed with GitHub closing keywords in commit
  messages when completed work lands on the default branch:
  https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/linking-a-pull-request-to-an-issue
- Issues can also be closed manually after work is complete or when work is not
  planned:
  https://docs.github.com/en/issues/tracking-your-work-with-issues/administering-issues/closing-an-issue
