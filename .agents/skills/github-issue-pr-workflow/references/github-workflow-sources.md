# GitHub Workflow Sources

- GitHub Issues track ideas and work:
  https://docs.github.com/en/issues/tracking-your-work-with-issues
- Sub-issues break larger issues into tasks and preserve parent/child tracking:
  https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/adding-sub-issues
- Branches connected to issues appear in the issue development section and PRs
  created from those branches are linked to the issue:
  https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/creating-a-branch-for-an-issue
- Pull requests propose changes from a branch so the default branch contains
  finished and approved work:
  https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request
- PRs can be linked to issues manually or with closing keywords. Linked issues
  auto-close when the linked PR is merged into the default branch:
  https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/linking-a-pull-request-to-an-issue
- Issues can also be closed manually after work is complete or when work is not
  planned:
  https://docs.github.com/en/issues/tracking-your-work-with-issues/administering-issues/closing-an-issue
